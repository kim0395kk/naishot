# 행정업무운영 편람 - 공문서 작성 표준 지침
# 이 파일을 수정하면 공문서 컴파일러가 자동으로 적용합니다.
# 마지막 수정: 2026. 1. 21.

## 1. 항목 체계 (Item Hierarchy)

공문서의 번호 체계는 다음 순서를 따른다:
1. → 가. → 1) → 가) → (1) → (가)

예시:
1. 첫 번째 대항목
   가. 첫 번째 중항목
      1) 첫 번째 소항목
         가) 세부 항목
            (1) 상세 항목
               (가) 최하위 항목

## 2. 문체 규범 (Style Rules)

다음 표현들을 행정 간결체로 변환한다:

| 원문 | 변환 |
|------|------|
| ~하고자 합니다 | ~하고자 함 |
| ~해 주시기 바랍니다 | ~협조 바람 |
| ~드립니다 | ~함 |
| ~하겠습니다 | ~할 예정임 |
| ~부탁드립니다 | ~바람 |
| ~입니다 | ~임 |
| ~습니다 | ~음 |
| ~주십시오 | ~바람 |
| ~감사합니다 | (삭제 또는 생략) |

## 3. 날짜 표기법 (Date Format)

- 연월일: YYYY. M. D. (예: 2026. 1. 21.)
- 마침표 필수
- 시간: HH:MM 형식 (예: 14:00, 09:30)
- 오전/오후 표기 금지, 24시간제 사용

## 4. 본문 구조 (Document Structure)

표준 공문서 구성:
1. 제목
2. 개요 (목적, 배경)
3. 근거 (법령, 규정, 지침)
4. 세부 내용 (본론)
5. 행정 사항 (협조사항, 기한, 담당자)
6. 끝.

## 5. 본문 마무리 (Ending)

- 본문의 마지막에는 반드시 "끝." 표기
- "끝." 앞에는 빈 줄 없음
- 별첨이 있는 경우: "별첨: OOO 1부. 끝."

## 6. 수신 표기 (Recipient)

- 수신: 수신자 참조
- 참조: 해당 부서/기관명
- 발신: 발신기관장 직위

## 7. 금지 표현 (Prohibited Expressions)

다음 표현은 공문서에서 사용하지 않는다:
- 감정적 표현 (매우, 정말, 꼭)
- 비격식 표현 (~해요, ~거예요)
- 외래어 남용 (가능하면 한글로)
- 중복 표현 (다시 재검토, 먼저 선점)

## 8. 개인정보 보호 (Privacy Protection)

다음 정보는 마스킹 처리:
- 성명: OOO
- 연락처: ***-****-****
- 주소: OO시 OO구 ***
- 주민등록번호: ******-*******
- 차량번호: **가 ****

## 9. 경어법 (Honorific Usage)

- 대민 문서: 존칭 사용 가능
- 대내 문서: 간결체 사용
- 기관간 문서: 격식체 사용
